
طلبك يحتوى على 5 ميزات كبيرة. سأنفذها بالترتيب التالى، كل واحدة فى رد منفصل عشان لو حصل خطأ نقدر نصلحه بسرعة:

## 1) إصلاح تصدير PDF فى فحص الأعراض  
المشكلة الحالية: `window.print()` يفتح نافذة جديدة لكن بعض المتصفحات (خاصة الموبايل) تمنع النوافذ المنبثقة فتفشل العملية بصمت.  
الحل: استخدام مكتبة `jspdf` + `html2canvas` (نصدّر مباشرة من نفس الصفحة بدون نافذة منبثقة) مع دعم Unicode عبر تضمين خط Cairo. زر "تصدير PDF" يحمّل الملف فوراً.

## 2) صفحات منفصلة لكل عنصر هيدر  
حالياً 20+ رابط فى الهيدر لكنها كلها صفحات موجودة بالفعل. سأضيف الناقص فقط:  
- `/about` — عن التطبيق  
- `/features` — المميزات  
- `/how-it-works` — كيفية الاستخدام  
- `/privacy` — الخصوصية  
كل صفحة لها `head()` مع title/description/og خاصة بها، ومحتوى مترجم بكل اللغات الـ14.

## 3) ترجمة كاملة لبيانات الأمراض  
المشكلة: بيانات الأمراض موجودة بـ`ar`/`en` فقط، باقى اللغات تعرض إنجليزى.  
الحل العملى (بدون تضخيم الملف 14×):  
- إضافة proxy `/api/public/translate` يستخدم Google Translate المجانى  
- مكون `<T>` يترجم النص حسب لغة المستخدم ويخزّن النتيجة فى `localStorage` (لا طلبات متكررة)  
- يُطبَّق على: اسم المرض، الوصف، الأعراض، الأسباب، العلاج، الإسعافات، الأدوية، كابشن كل قسم  
- بعد أول زيارة تصبح كل البيانات متاحة offline تلقائياً (مخزَّنة فى الكاش)

## 4) لغة الجهاز تلقائياً عند أول دخول  
موجود جزئياً فى `I18nProvider`. سأحسّنه ليدعم: `navigator.languages` (قائمة كاملة)، تطابقات إضافية (zh-CN، pt-BR، إلخ)، وتنبيه صغير عند أول زيارة يخبر المستخدم باللغة المكتشفة ويعطيه خيار التغيير.

## 5) تنقّل صوتى محسّن بأوامر مرنة  
حالياً يطابق نصياً فقط. سأحسّنه ليستخدم:  
- مصفوفة مرادفات لكل صفحة بكل اللغات (مثلاً "أدوية"، "علاج"، "دواء"، "medicines"، "pharmacy" → `/medications`)  
- أوامر إضافية: "ارجع"، "اقرأ"، "اوقف"، "أعد"، "افتح القائمة"  
- مطابقة ضبابية (fuzzy) بدل المطابقة الحرفية

**ملاحظات:**  
- ترجمة Google عبر proxy مجانية لكن لها حدود طلبات؛ التخزين فى localStorage يحلّ معظم الاستخدام  
- ستضاف 4 صفحات هيدر جديدة فقط — الباقى موجود  
- تصدير PDF سيستخدم الخط المضمَّن داخل الـ PDF حتى يعمل offline
